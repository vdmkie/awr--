Привет 
Помоги создать красивое полноценное и функциональное мини приложение для телеграмм и веб версии для ПК (что б можно было с ПК использовать). 
приложение должно быть с современным дизайном , логотипом (картинкой) при запуске, кнопками, меню. 

Название приложения:
AWR 
assistant work resurs 

Суть приложения: 
Есть супер-админ (1 человек) 
Есть админы ( 2 человека) 
Есть бригады (2 бригады по 2 чел.) 
Есть кладовщик (1 человека)

Афторизация: 
пользователь открывает бот, нажимает поледиться номером, если номер есть в базе то бот дает кнопку запустить приложение, если номера нет в базе - то пишет в доступе отказано. 
После того как бот принял номер и пользователь нажал запустить приложение - На стартовом экране открывается логотип + пункт авторизоваться (Логин (тот же номер который бот принял) + пароль (вноситься в базу)).  Пользователь авторизируется и, если он есть в базе то ему открывается мини приложение с его интерфейсом в зависимости кем он является (админ, бригада, кладовщик) 



Супер-админ: 
- назначает задачи админам 
- может назначать задачи бригадам 
- видят общий список задач (с фильтрами по боигадам), 
- видят остатки материалов и инструмента по бригадам. 
- Могут переназначить в задании бригаду, удалить задание, менять статус задания.
- есть пункт отчёты (где хранятся все отчёты бригад и есть фильтры для поиска отчётов по статусу, адресу, бригаде. 
- есть пункт "доступ инфо" (где хранится инфо о доступе из отчётов бригад и есть тоже фильтр по адресу.
- есть пункт "дома на сдачу" (куда вносятся адреса домов)

Админы:
- Назначают на бригады задания. 
- видят общий список задач (с фильтрами по боигадам), 
- видят остатки материалов и инструмента по бригадам. 
- Могут переназначить в задании бригаду, удалить задание, менять статус задания.
- есть пункт отчёты (где хранятся все отчёты бригад и есть фильтры для поиска отчётов по статусу, адресу, бригаде. 
- есть пункт "доступ инфо" (где хранится инфо о доступе из отчётов бригад и есть тоже фильтр по адресу.
- есть пункт "дома на сдачу" (куда вносятся адреса домов)    

задание должно иметь поля:
- Адрес
- колличество этажей и парадных 
- вид работ (воздушка, перемычка, растянуть дом, дом под ключ, бурение, сварки) 
- ТЗ
- Доступ 
- пометка 
- Бригада что назначена. 

Есть статусы для заданий. бригады не могут менять статус задания. 
- Новая задача (не имеет бригаду исполнителя, и только после перевода в статус "в работе" даёт возможность назначить бригаду" 
- В работе 
- Выполнено 
- Отложено
- Проблемный дом 


Бригады:
- видят список задач назначенные им (в работе).
- видят список выполненных задач
- видят список отложенных задач 
- есть пункт "доступ инфо" (где хранится инфо о доступе из отчётов бригад и есть тоже фильтр по адресу.
- видят остаток материалов и инструмента что закреплён за ними. 
- выполняют задачи, дают отчёт и списывают материал по каждой задаче.
- отчет должен быть из 4 частей, и может заполняться в разное время
1 часть - комментарий 
2 часть - доступ
3 часть - Фото
4 часть - списать материалы
после заполнения каждого из пунктов отчёта (при сохранении) появляется галочка. 

задача должна оставаться в статусе "в работе" пока не будут сделан отчет по 4 пунктам. после переводиться в статус "выполнено". 


Кладовщик:
имеет меню "мой склад" 
- добавляет материалы и инструмент на общий склад 
- может списывать материалы и инструмент со склада (при списании пишет комментарий). 
- выдает бригадам материалы и инструмент, может вернуть на склад с бригады, переместить между бригадами
- ведёт учёт.
- Видит список бригад, у кого сколько материалов на руках 
- видит список бригад , у кого какой инструмент, 
- видно общие остатки по складу. 
Есть кнопка выгрузить отчёт в excel файл.

Материалы считаются в метрах ,штуках и килограммах (м, шт и кг)

Список материалов и в чем измеряются: 
- Кабель ВОК 4 100(м)
- БО/16 100(шт)
- шпаклёвка 100(кг)

Список Инструмента должен фиксироваться по серийному номеру:
Список:
перфоратор проводной
перфоратор безпроводной  

 
У меня в ТГ есть супергруппа с топиками для каждой бригады. 
Хочу что б при подаче отчетов - отчеты сохранялись в базе и + отправлялись в супер группу телеграмм (у каждой бригады свой топик) 

Весь процесс должен быть именно в приложении , телеграм бот только открывает мини приложение

У всех пользователей должно быть меню с кнопками . 


Приложение буду запускать на рендер через гитхаб
